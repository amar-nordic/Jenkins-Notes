# Groovy for Jenkins Pipelines

Groovy (a programming language) is used for Jenkins pipelines and has a syntax similar to Java but with added productivity features.

- Jenkins Pipelines are defined using a Groovy-based Domain-Specific Language (DSL).
- Groovy combines features of Java with scripting capabilities
- Groovy has scripting capabilities - It can execute scripts without the need for compilation
- It's widely used for automation, scripting, and building applications, especially in environments like Jenkins and Gradle
- Excellent for scripting tasks, test automation, and building Domain-Specific Languages (DSLs).
- Groovy is a dynamically typed language by default, but it offers flexibility to use static typing when desired
- Groovy works seamlessly with Java and is a very popular choice for Java developers
- Syntax of Groovy is similar to Java
- Most valid Java files are also valid Groovy files
- The keyword **def** is used in Groovy for dynamic typing
